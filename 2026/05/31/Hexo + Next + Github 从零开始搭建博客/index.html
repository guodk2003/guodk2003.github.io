前言

之前许多课程的实验都参考往年的学长的博客，从那时起便萌生了搭建自己博客的念头，记录下自己的学习过程。终于，前段时间选择用 Hexo 搭建了自己的博客，历经了一个多月断断续续的美化，整体效果已符合心中预期😇，特此记录整个过程。

一、Hexo 博客的搭建

先贴下 Hexo 官方文档，感兴趣的朋友可直接参考官方指南。

环境准备

在安装 Hexo 之前得安装以下两个程序

  1. Node.js：Hexo 本身就是用 Node.js 写的，执行 hexo generate 时，Node.js 把 Markdown 文章变成 HTML 页面。
  2. Git：用 Git 工具把博客部署到 GitHub 上。

安装 Node.js

安装 Node.js 时会自动包含 npm 包管理器，无需单独安装。

访问Node.js官网，安装 LTS 版（长期维护版）即可。

配置环境变量：打开我的电脑->右键->属性->高级系统设置->高级->环境变量，将 nodejs 加入环境变量。

打开 cmd 命令窗口，执行下面的命令，如果能查询到版本号，那么就算安装成功。

访问 Git 官网，下载并安装git。

  • source：页面与文章目录，内部的_posts文件夹里存放着你的文章
  • themes：主题目录
  • public：存放生成的静态网页，部署就是把里面的内容复制到云端
  • _config.yml：此博客的配置文件（注意不要和之后的主题配置文件搞混）

接下来介绍几个 Hexo 命令。

  • hexo server：启动预览服务器（实时更新，刷新网页即可）
  • hexo clean：清空缓存和public目录，遇到一些问题时可以尝试运行此命令
  • hexo generate：在public目录中生成静态网页文件
  • hexo deploy：从_config.yml中读取设置部署至远程站点。
  • hexo new "My New Post"：在./source/_posts中创建新文章，引号内写标题，也可以选择不使用此命令自己手动新建文件

配置 Github

目前本地已经可以访问到自己的博客了，但为了让所有人都可访问，还需部署到 Github上去。

创建 Github 仓库

打开 GitHub，点击右上角的加号->New repository来新建仓库，仓库必须名称为你的用户名.github.io。

回到 git bash黑窗口中，输入以下两个命令（逐条）：

yourname改为你的GitHub的用户名

youremail改为你的注册GitHub时的邮箱

使用ssh-keygen生成密钥。根据提示完成填写。

之后打开.ssh文件夹，会发现有id_rsa（私钥）和id_rsa.pub`（公钥）两个文件

将公钥添加到 GitHub，打开GitHub，点击右上角头像->Setting->左侧 SSH and GPG keys->New SSH key

复制 id_rsa.pub 文件里的内容，取一个title如 blog-key。然后点击Add SSH key即可。

kimi 牛逼😀

但如果图片或其父元素带有 style="zoom:25%" 等 CSS zoom 属性，Medium Zoom 会失效，导致图片无法正常放大，这是 NexT 的已知 Bug。但我用Typora 写博客，Typora 缩放图片会自动插入 style="zoom:xx%"。
